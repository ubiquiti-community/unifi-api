# OpenAPI Generator configuration

This directory is **output #2** of `cmd/fields`: the configuration that lets
OpenAPI Generator build UniFi API clients in any supported language while
reproducing the naming cleanup go-unifi applies.

## Files

| File | Source | Applies to | Purpose |
|------|--------|-----------|---------|
| `model-name-mappings.cfg` | **generated** | all languages | `SchemaName=SchemaName` — locks canonical model names / acronym casing (`RADIUSProfile`, `ClientGroup`, `Network`) so the generator doesn't re-case them. |
| `go-name-mappings.cfg` | **generated** | Go only | `json_name=GoFieldName` — the exact go-unifi field cleanup (`dhcpd_enabled=DHCPDEnabled`, `wan_networkgroup=WANNetworkGroup`). |
| `languages/<lang>.yaml` | static | one language | generator name + per-language `additionalProperties`. |
| `templates/` | static | opt-in | mustache template overrides (see `templates/README.md`). |

The two `.cfg` files are regenerated by `go generate ./...` (i.e. `go run
./cmd/fields -latest`) directly from the UniFi field definitions, so they never
drift from the spec.

### Why property mappings are Go-only

go-unifi's cleanup produces **Go** identifiers (`DHCPDEnabled`). Forcing those on
every language would yield non-idiomatic clients (a Python attribute called
`DHCPDEnabled`). The spec already carries clean `snake_case` property names, so
every non-Go generator applies its own idiomatic casing on top of that clean
base. Only the **model names** are shared across languages. To opt a language
into Go-style property names, pass its `--name-mappings` the same way the Go
target does in the `Makefile`.

## Generating a client

From the repo root (requires Docker):

```bash
make client-go          # one language
make clients            # all curated languages -> clients/<lang>/
```

The `Makefile` invokes `openapitools/openapi-generator-cli`, passing
`languages/<lang>.yaml` as the config plus the generated mapping files and the
shared `--openapi-normalizer` rules.

## Adding a language

1. Add `languages/<lang>.yaml` (`generatorName:` + any `additionalProperties:`).
2. Add `<lang>` to `LANGS` in the root `Makefile` and the matrix in
   `.github/workflows/clients.yml`.
